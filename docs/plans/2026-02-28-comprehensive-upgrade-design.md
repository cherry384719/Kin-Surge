## 2. 闯关系统

### 数据模型改造

```sql
-- dynasties 表新增
ALTER TABLE dynasties ADD COLUMN unlock_requirement int DEFAULT 0;

-- poets 表新增
ALTER TABLE poets ADD COLUMN sort_order int DEFAULT 0;
ALTER TABLE poets ADD COLUMN is_boss boolean DEFAULT false;

-- progress 表改造
-- 新增 poet_id, stars, completed 字段
```

### 解锁规则

1. 汉朝（第一朝代）默认解锁
2. 朝代内第一个诗人默认解锁，通关后解锁下一个
3. 所有普通诗人通关后，解锁BOSS关
4. BOSS关通过后，解锁下一个朝代

### 星级评价

- 3星: 全部答对，无提示
- 2星: 有1-2次答错后重试
- 1星: 使用了"查看答案"

### BOSS关

从该朝代所有诗人的诗中随机抽5题混合出题，需全部答对。

## 3. 页面设计

### 登录/注册页
- 水墨山水背景 + 半透明卡片
- "通天路" 毛笔字体标题 + "诗词闯关之旅" 副标题
- 邮箱(格式校验，不验证) + 密码 + 显示名称
- 登录/注册 tab 切换

### 朝代地图页（首页）
- 横向卷轴布局，6朝代从左到右
- 朝代为"印章/门"样式
- 已解锁: 彩色+朝代配色+星数进度
- 未解锁: 灰色+锁图标
- 顶栏: 用户名、金币、主题切换

### 诗人列表页
- 竖向"名帖"卡片
- 诗人名、朝代、星级
- BOSS关在底部，金色边框特殊样式

### 答题页
- 书卷/对联风格
- 上方: 诗人给出的诗句 + 诗人头像占位
- 下方: 输入框
- 反馈: 绿色(对) / 黄色(接近) / 红色(错)
- 底部进度条

### 结算页
- 星级评价 + 星星动画
- 金币获得数量
- "继续下一关" / "返回"

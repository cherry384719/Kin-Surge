# 通天路 游戏优化设计文档

日期: 2026-03-01
状态: 已确认

## 概述

分三个阶段优化游戏体验：新题型与流程优化 → 数据层整理 → 游戏化扩充。

---

## 阶段 1：新题型 + 流程优化

### 1.1 新挑战题型

现有题型仅"填下一句"。新增 3 种题型，形成难度梯度：

```
选择题 (beginner) → 填空题 (intermediate) → 排序题 (intermediate+) → 飞花令 (master)
```

**选择题（初级）**
- 给出一句诗，从 4 个选项中选出正确的下一句
- 错误选项从同诗人/同朝代的其他诗句中随机抽取
- 适合初次遇到的诗词

**排序题（中级+）**
- 给出一首诗的 4 句（打乱顺序），玩家拖拽排列成正确顺序
- 每首诗整体作为一道题
- 适合已经通过选择题的诗词

**飞花令（高级/特殊模式）**
- 给出一个关键字（如"花""月""春"）
- 玩家轮流输入包含该字的诗句
- AI 侧从数据库中匹配含该字的诗句应答
- 限时制，答不出来就结束

### 1.2 游戏流程优化

**诗人对话引入**
- 进入挑战前显示诗人简短对白（2-3 句），介绍背景
- 例如：李白说"吾乃诗仙李白，你可知我那首《静夜思》？来，接我一招！"
- poets 表新增 `challenge_intro` 字段

**关卡结构（每个诗人 3 轮）**
- 第 1 轮：选择题（2-3 道）
- 第 2 轮：填空题（2-3 道）
- 第 3 轮：排序题（1-2 道）
- 每轮结束有小结算（显示本轮得分）
- 全部通过后显示总结算 + 星级评定

**结算优化**
- 结算页显示：总得分、星级、获得金币、新解锁内容
- 首次通关有额外奖励动画
- 3 星通关有特殊庆祝效果

**进度可视化**
- 朝代地图显示每个朝代的完成百分比
- 诗人页面显示已完成/总挑战数
- 增加总体进度条（已掌握诗词数 / 总诗词数）

## 阶段 2：数据层整理

### 2.1 新增数据库表

| 表名 | 用途 |
|------|------|
| `challenge_sessions` | 每次挑战的完整记录（诗人、题型、得分、时长） |
| `challenge_answers` | 每道题的作答记录（题型、正确/错误、用时） |
| `daily_tasks` | 每日任务配置和完成状态 |
| `user_streaks` | 连续登录数据（从 localStorage 迁移） |
| `user_energy` | 能量数据（从 localStorage 迁移） |

### 2.2 修改现有表

| 表名 | 变更 |
|------|------|
| `poets` | 新增 `challenge_intro` 字段（诗人开场白） |
| `poem_lines` | 新增 `difficulty_tier` 字段（适合哪个题型） |
| `user_profiles` | 新增 `total_poems_mastered`、`total_challenges_completed` |
| `poet_progress` | 新增 `rounds_completed`（记录每轮完成情况） |

### 2.3 localStorage → Supabase 迁移

- `kin-streak` → `user_streaks` 表
- `kin-energy` → `user_energy` 表
- `kin-daily-bonus` → `daily_tasks` 表
- 保留 `kin-challenge-{poetId}` 作为会话缓存（临时数据无需持久化）

## 阶段 3：游戏化扩充

### 3.1 每日任务系统

- 每天 3 个随机任务：
  - "完成 1 次挑战"
  - "连续答对 5 题"
  - "挑战唐朝的诗人"
- 完成任务奖励金币 + 经验
- 全部完成有额外奖励

### 3.2 排行榜

- 激活已有的 `scores` 表
- 3 种排行：日榜、周榜、总榜
- 按总得分排名
- 显示前 20 名 + 自己的排名

### 3.3 段位系统激活

- 6 级段位：童生 → 秀才 → 举人 → 进士 → 翰林 → 诗仙
- 根据累计得分/掌握诗词数升级
- 每个段位有对应图标和称号

### 3.4 更多诗词内容

- 从当前 65 首扩充到 150+ 首
- 补充每个朝代的经典作品
- 确保覆盖小学到高中的课程标准
